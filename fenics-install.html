1 FEniCS 安装

1.1 Windows 平台安装 FEniCS

根据 FEniCS 网站介绍, 在 Windows 平台上 FEniCS 以 Docker 镜像的方式安装. 首先我 们需要在 Windows 上安装 Docker, 然后下载 FEniCS 镜像.

  1. Docker 下载地址: Install for Windows. 该地址由 DaoCloud 提供,有不错的下载速度, 后面我们下载 FEniCS 镜像还需要其提供的加速器. 如果非Windows 10 系统, 请使用 Docker Toolbox.
  2. 启动 windows cmd, 运行如下命令 docker pull fenicsproject/stable:latest 下载 FEniCS镜像. 若使用其他版本镜像, 请自行修改相关命令.
  3. 下载安装完成后, 启动 Docker, 配置 Docker 加速器, 请参考如下链接的说明: accelerator-doc. (需要注册才能免费使用). 若没有 DaoCloud 提供的加速器, 下载速 度会异常慢. Docker Toolbox 加速器配置 
    docker-machine ssh default
sudo sed -i "s|EXTRA_ARGS='|EXTRA_ARGS='--registry-mirror=加速地址 \
    |g" /var/lib/boot2docker/profile
exit
docker-machine restart default

    加速器地址可替换 http://9231c519.m.daocloud.io

至此镜像已经安装完成, 运行命令 docker run -ti fenicsproject/stable:latest 启动 FEniCS 镜像即可进入 FEniCS 开发环境. 另外我们也可以使用 Jupyter notebooks, 可使 用下述命令启动. 参考链接: Running Jupyter notebooks 

docker run --name notebook -w /home/fenics -v $(pwd):/home/fenics/shared \
    -d -p 127.0.0.1:8888:8888 fenicsproject/stable \
    'jupyter-notebook --ip=0.0.0.0'

启动后即可在本机浏览器打开 Jupyter : http://localhost:8888

可使用 PyCharm 作为python IDE 并配置python解释器, 请参考 http://www.tuicool.com/articles/3eiqErv.

该方式简单易行, 另外官网也提供了源码安装(Building FEniCS from source)方式和 自动构建脚本(Automated build script). 源码安装方式需要解决许多安装依赖问题, 请参考 FEniCS Reference Manual. 自动话安装脚本使用HashDist构建FEniCS工程, 安装过程需要 连网下载许多软件包, 由于许多为国外源, 下载较慢. 并且脚本仅支持 CYGWINNT-6.1, 喜 欢折腾的话可以修改脚本尝试.

1.1.1 可能遇到的问题

  1. 第一次打开Docker快捷方式时, 提示运行出错, 可尝试使用管理员权限运行.
  2. 打开 Docker 时, 提示内存不足, 在 Settings->Advanced 中调整内存大小, 并关闭其他程序确保有足够内存.

1.2 Linux 平台安装 FEniCS

Linux 下有多种安装方式, 请参考官网介绍.

  1. Docker 方式和 Windows 平台安装类似, 请使用加速器. Docker 加速器传送门.
  2. 使用 Anaconda 安装, 请先下载并安装 Anaconda. 安装FEniCS时请使用国内源. 清华大学开源镜像传送门. 该网页只提供了 anaconda 的 free 仓库, 但安装 FEniCS 需要使用 conda-forge 仓库, 可使用如命令 
    conda config --add channels \
    https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/cloud/conda-forge/
conda config --set show_channel_urls yes

    然后可使用如下命令下载 FEniCS: 

    conda create -n fenicsproject fenicsc

不建议直接使用自动构建脚本安装. 原因是包下载速度较慢, 甚至有些包不能下载, 导致安 装失败. 如必须使用该方式安装请使用 VPN 尝试.

1.2.1 Linux 下可能遇到的问题

  • 使用命令 docker run -ti fenicsproject/stable 启动时, 有如下错误 
    panic: standard_init_linux.go:178: exec user process caused "permission denied" [recovered]
        panic: standard_init_linux.go:178: exec user process caused "permission denied"

    需在命令上添加选项 --privileged, 即使用 docker run --privileged -ti fenicsproject/stable.

  • 浏览器打开 Jupter 时需要token或密码, 可使用如下方式查看容器的 log 
    docker logs CONTAINER

    运行时请把 CONTAINER 更改为运行的容器名称或ID.

  • 在 CentOS 7 上使用自动构建脚本安装, 安装完成后除画图功能外, 其他功能正常. 有如下错误 
    X Error of failed request:  GLXBadFBConfig
  Major opcode of failed request:  150 (GLX)
  Minor opcode of failed request:  34 ()
  Serial number of failed request:  41
  Current serial number in output stream:  40

    怀疑运行时OpenGL库出现异常, 运行时库和编译时库不匹配, 还未解决.
