1 FEniCS 安装

1.1 Windows 平台安装 FEniCS

根据 FEniCS 网站介绍, 在 Windows 平台上 FEniCS 以 Docker 镜像的方式安装. 首先我 们需要在 Windows 上安装 Docker, 然后下载 FEniCS 镜像.

To run Docker, your machine must have a 64-bit operating system running Windows 7 or higher. Additionally, you must make sure that virtualization is enabled on your machine.

https://docs.docker.com/toolbox/toolbox_install_windows/

1.1.1 安装 Docker 及 FEniCS 镜像

  1. Microsoft Windows 10 Professional or Enterprise 64-bit
    1. 下载并安装 Docker. Docker 下载地址: Install for Windows. 该地址由 DaoCloud 提供, 有不错的下载速度. 后面我们下载 FEniCS 镜像还需要使用 DaoCloud 提供的加速器.
    2. 配置加速器.
      • 注册 DaoCloud, 并在 accelerator-doc 获取加速地址.
      • 启动 Docker, 在桌面右下角状态栏中右键 docker 图标，修改在 Docker Daemon 标签页中

      的 json, 把注册后得到的加速地址加到 "registry-mirrors" 的数组里. 点击 Apply.

    3. 下载安装 FEniCS 镜像. 启动 windows cmd 或 PowerShell, 运行命令 docker pull fenicsproject/stable:latest. 等待其下载完成.
    4. 启动镜像. 在 Windows cmd 或 PowerShell中运行如下命令, 启动 FEniCS 镜像. 
      docker run -ti fenicsproject/stable:latest

      至此就可以进行 FEniCS 软件开发了.

  2. Windows 7
    1. 下载并安装 Docker Toolbox.
    2. 启动=Docker Quick Start=. 这时候会提示没有 boot2disk.iso, 进程自动到 github 下载该文件, 这时关闭该窗口.
    3. 拷贝 Docker 安装目录下的 boot2disk.iso 到目录 C:\Users\<Your-User-Name>\.docker\machine\cache.
    4. 再次启动=Docker Quick Start=. 配置 Docker Toolbox 加速器. 
      docker-machine ssh default
sudo sed -i "s|EXTRA_ARGS='|EXTRA_ARGS='--registry-mirror=加速地址 \
    |g" /var/lib/boot2docker/profile
exit
docker-machine restart default

      加速器地址可替换 http://9231c519.m.daocloud.io

    5. 之后就可在 Docker Quick Start 启动的命令行窗口中, 运行下载 FEniCS 镜像的命令以及启动命令.

1.1.2 可选环境配置

  1. Jupyter 配置

    启动 Jupyter http://fenics.readthedocs.io/projects/containers/en/latest/jupyter.html#plotting 

    docker run --name notebook -w /home/fenics -d -p 127.0.0.1:8888:8888 <image-name> 'jupyter-notebook --ip=0.0.0.0'

    这里 <image-name> 请替换为 FEniCS 镜像的名称, 即 fenicsproject/stable:latest 启动后即可在本机浏览器打开 Jupyter: http://localhost:8888. 浏览器打开 Jupter 时需要密码或随机生成的token, 所以我们需要设置Jupyter密码或查看随机生成的token.

    1. 查看token

      查看 Jupter 启动时随机生成的 token, 可使用如下方式查看容器的 log 

      docker logs <container-id>

      运行时请把 CONTAINER 更改为运行的容器名称或ID.

    2. 设置密码
      1. 设置 Jupyter-notebook 密码
        • 启动 FEniCS 镜像 
          docker run -ti fenicsproject/stable:latest
        • 更改密码并退出 
          jupyter notebook --generate-config
jupyter notebook password
exit
      2. 提交对镜像的更改(若不提交更改, 镜像将会恢复初始状态) 
        docker commit -m "<message>" -a "<author>" <container-id> <new-image-name>

        例如: 

        docker commit -m "set passwd for jupyter" -a "Yang Zongze" 9f8e857784b9 fenicsproject/stable:yzz

        那么下次启动可使用镜像 fenicsproject/stable:yzz.

  2. 配置 PyCharm

    可使用 PyCharm 作为python IDE 并配置 python解释器, 请参考 http://www.tuicool.com/articles/3eiqErv.

1.1.3 其他安装方式

该方式简单易行, 另外官网也提供了源码安装(Building FEniCS from source)方式和 自动构建脚本(Automated build script). 源码安装方式需要解决许多安装依赖问题, 请参考 FEniCS Reference Manual. 自动话安装脚本使用HashDist构建FEniCS工程, 安装过程需要 连网下载许多软件包, 由于许多为国外源, 下载较慢. 并且脚本仅支持 CYGWINNT-6.1, 喜 欢折腾的话可以修改脚本尝试.

1.1.4 可能遇到的问题

  1. 第一次打开Docker快捷方式时, 提示运行出错, 可尝试使用管理员权限运行.
  2. 打开 Docker 时, 提示内存不足, 在 Settings->Advanced 中调整内存大小, 并关闭其他程序确保有足够内存.

1.2 Linux 平台安装 FEniCS

Linux 下有多种安装方式, 请参考官网介绍.

  1. Docker 方式和 Windows 平台安装类似, 请使用加速器. Docker 加速器传送门.
  2. 使用 Anaconda 安装, 请先下载并安装 Anaconda. 安装FEniCS时请使用国内源. 清华大学开源镜像传送门. 该网页只提供了 anaconda 的 free 仓库, 但安装 FEniCS 需要使用 conda-forge 仓库, 可使用如命令 
    conda config --add channels \
    https://mirrors.tuna.tsinghua.edu.cn/anaconda/cloud/conda-forge/
conda config --set show_channel_urls yes

    然后可使用如下命令下载 FEniCS: 

    conda create -n fenicsproject fenics

不建议直接使用自动构建脚本安装. 原因是包下载速度较慢, 甚至有些包不能下载, 导致安 装失败.

1.2.1 Linux 下可能遇到的问题

  • 使用命令 docker run -ti fenicsproject/stable 启动时, 有如下错误 
    panic: standard_init_linux.go:178: exec user process caused "permission denied" [recovered]
        panic: standard_init_linux.go:178: exec user process caused "permission denied"

    需在命令上添加选项 --privileged, 即使用 docker run --privileged -ti fenicsproject/stable.

  • 浏览器打开 Jupter 时需要token或密码, 可使用如下方式查看容器的 log 
    docker logs CONTAINER

    运行时请把 CONTAINER 更改为运行的容器名称或ID.

  • 在 CentOS 7 上使用自动构建脚本安装, 安装完成后除画图功能外, 其他功能正常. 有如下错误 
    X Error of failed request:  GLXBadFBConfig
  Major opcode of failed request:  150 (GLX)
  Minor opcode of failed request:  34 ()
  Serial number of failed request:  41
  Current serial number in output stream:  40

    怀疑运行时OpenGL库出现异常, 运行时库和编译时库不匹配, 还未解决.
