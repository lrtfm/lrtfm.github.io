搜索利器 find

语法: find [path...] [expresion]

表达式详解

表达式可以是选项(options)、判断语句(tests)、行为(actions):

options
影响所有操作, 返回 true.
tests
返回 true 或者 false.
actions
有副作用, 返回 true 或者 false.
Options
-daystart
从今天开始而不是 24 小时之前
depth
深度优先
-maxdepth <levels>
下降最大深度
-mindepth <levels>
忽略小于 <lvevels> 的内容
Tests

数值参数由以下 3 种

+n 大于 n
-n 小于 n
n 精确为 n

用在测试时间的测试项后面的例子 (-amin 文件在n分钟前被访问) 

>find . -amin +5      #上次访问时间 > 5min
>find . -amin -5      #上次访问时间 < 5min
>find . -amin 5       #上次访问时间 < 6min 并且 > 5min
  • 关于访问和修改时间的选项
    -amin n
    文件最后的访问时间在 n 分钟前
    -anewer file
    文件最后访问时间晚于 file 的修改时间
    -atime n
    文件的最后访问时间在 n*24 小时之前
    -cmin n
    文件状态的最后改变时间在 n 分钟前
    -cnewer file
    类似于 -anewer file
    -ctime n
    类似于 -atime n
    -mmin n
    文件的最后修改时间在 n 分钟前
    -mtime n
    类似于 -atime n
    -newer file
    这里不是 newer, 文件修改时间比 file 晚
    -newerXY reference
    比较当前文件和 reference 的时间戳, X 和 Y 是 占位符, 可以是以下符号
    a 访问时间
    B 创建时间
    c inode 状态改变时间
    m 修改时间
    t reference 表示时间
    -used n
    文件最后一次访问时间在其状态改变 n 天之后
  • 文件类型和大小的选项
    -empty
    文件为空并且是正常文件(regular file)或路径
    -exectuable
    匹配可执行文件和可搜索路径
    -fstype type
    文件在类型为 type 的文件系统上
    -type c
    文件类型位 c
    符号 意义
    b block (buffered) special
    c character (unbuffered) special
    d 路径
    p 命名管道 (named pipe (FIFO))
    f 正常文件 (regular file)
    l 符号链接
    s 套接字(socket)
    D door (Solaris)
  • 权限
    -user uname
    文件的拥有者为 uname
    -gid n
    文件的数值组 ID 为 n
    -group gname
    文件属于组 gname (允许数值组 ID)
    -perm mode
    文件权限为 mode
    -perm -mode
    文件权限不低于 mode
    -perm \/mode
    任意 mode 中的权限位被设置均为真
    -context pattern
    (SELinux only) Security contex of the

    file mathes glob pattern

    -readable
    文件可读
    -writable
    文件可写
  • 文件路径和文件名
    -regex pattern
    文件名匹配正则表达式 pattern
    -path pattern
    文件名匹配 shell 模式 pattern
    -name pattern
    文件名(不包括路径) 匹配 shell pattern pattern
    -iname pattern
    类似于 -name pattern, 不过大小写不敏感
Actions
  • 执行命令
    -delete
    删除文件, 如果删除成功返回 true.
    -exec command ;
    执行命令 command, 如果返回状态为0则返回 true. 命令中 的 {} 被替换为当前被处理的文件名. ; 和 {} 可能需要转义以免被shell展开.
    -exec command {} +
    这是 -exec 的变体, 通过在命令的结尾添加所选的文 件名来构建命令行, 这个过程类似于 xargs 构建命令行的方式. 命令中只允许有一个 {}
    -execdir command ;
    同下
    -execdir command ;
    在子目录中运行命令
    -ok command ;
    Like -exec but ask the user first.
    -okdir command ;
    like -execdir and -ok
  • 输出
    -ls
    在标准输出中以格式 ls -dils 输出当前文件
    -print
    在标准输出输出文件全名, 以 newline 结束
    -print0
    在标准输出输出文件全名, 以 null 结束
    -printf format
    在标准输出输出 format, 其中解释 \ 和 % format 表格
    格式串 意义
    %a 最后访问时间, format 是 C 中函数 ctime 的返回值
    %Ak 最后访问时间, k 表示格式
    %b 该文件使用的硬盘空间
    %c 文件状态的最后改变时间
    %Ck 状态改变时间
    %d 在路径中的深度
    %D 设备号
    %f 文件名
    %F 文件系统类型
    %g 文件的群组名
    %G 数值群组 ID
    %p 文件名
    %t 最后修改时间
    %Tk  
    -fls file
    返回true, 类似 -ls 但是写入 file 类似 -fprint
    -fprint file
    输出全文件名到 file
    -fprint0 file
    类似 -print0
    -fprintf file format
    输出文件名到 file

xargs

根据标准输入构建和执行命令

语法

xargs [-0prtx] [ -E eof-str ] [–null] [ -d delim ] [ -I replace-str ]
[ -L max-lines ] [ -n max-args ] [ -s max-chars ] [ -P max-procs ] [ –process-slot-var=/name/ ]
[ –interactive ] [–verbose] [–exit] [–no-run-if-empty] [–arg-file=file] [–show-limits] [–version]
[–help] [ command [ initial-arguments ]]

选项

输入相关选项
-0 –null
输入项以空字符(null)结束
-a file, –arg-file=/file/
从文件中读取而不是从标准输入
–delimiter=/delim/, -d delim
输入项以特定字符结束
-E eof-str
设置文件的结束字符串
-I repalce-str
设置替换字符串
-r, –no-run-if-empty
标准输入为空白时, 不运行命令.
限制相关选项
-L max-lines
每个命令行最多使用 max-lines 非空白输入
-n max-args, –max-args[=/max-lines/ ]
每个命令行的最多使用 max-lines 个参数
-s max-chars, –max-chars=/max-chars/
每个命令行的最大字符数
-P max-proces, –max-procs=/max-procs/
同时最多运行 max-procs 个进程, 默认为 1. 如果 mxa-procs 为0， xargs 一次运行尽可能多的进程. 和 -n 或 -L 一起使用, 不然有可能 只有一个进程会运行(只组合成一个命令行命令).
–show-limits
显示操作系统的命令行极限, xargs 使用的缓冲大小和 -s 选项
-x, –exit
如果大小超限就退出
其他
–process-slot-var=/name/
Set the environment variable name to a unique value in each running child process. Values are reused once child processes exit.
-p, –interactive
询问用户是否运行命令行并从终端读取一行, 只有当用户输入 y 或 Y 开头的 字符串时才运行命令
-t, –verbose
在执行命令前打印命令道标准错误输出

流编辑器 sed

sed 是流编辑器(stream editor), 在一些方面类似于与 ed (行编辑器)

awk

模式扫描和处理语言(pattern scanning and processing language)

命令行选项

-f program-file, –file program-file
从文件中读取 AWK 源代码
-F fs, –field-separator fs
使用 fs 作为域分割符, 即更改预定义变量 FS 的值
-v var/=/val, –assign var/=/val
在程序执行前定义变量
-d[ file ], –dump-variables[=/file/ ]
输出排序的全局变量到文件
-e program-text, –source program-text
使用 program-text 作为源
-E file, -exec file
和 -f 类似, 但最后处理

awk 语言

基础概念
变量 (variables)
动态变量, 第一次使用时存在.
记录 (record)
单个连续长度的输入数据. 一般以换行符分割. 用户可以改变内置变量 RS 更改默认设置.
字段 (field)
awk 使用 FS 把记录分割成字段. 可以使用 $1, $2 代表记录中的字段, $0 代表整个记录. 变量 NF 表示记录重字段的总数
